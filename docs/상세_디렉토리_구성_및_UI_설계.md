## 4. 상세 UI 상호작용

### 4.1 매물 등록 및 관리 

1. **매물 목록 화면에서**:
   - 사용자는 필터 컴포넌트를 통해 매물 검색 가능
   - 매물 카드를 클릭하여 상세 정보 확인 가능
   - "매물 등록" 버튼을 클릭하여 새 매물 등록 폼으로 이동

2. **매물 등록/수정 폼에서**:
   - 모든 필수 필드 입력
   - 매물종류, 진행상태, 거래유형은 드롭다운으로 선택
   - 담당자 배정 및 소유주 연결
   - 공동중개사 추가 가능
   - 사진/영상 업로드 기능
   - "저장" 버튼을 클릭하여 매물 정보 저장

3. **매물 상세 화면에서**:
   - 모든 매물 정보 확인
   - "수정" 버튼을 클릭하여 매물 정보 수정
   - "매칭 관리" 탭을 통해 담당자, 공동중개사 관리

### 4.2 매칭 관리

1. **담당자 배정**:
   - 현재 담당자 확인
   - 새 담당자 선택 및 변경 사유 입력
   - 담당자 변경 이력 확인

2. **공동중개사 관리**:
   - 현재 공동중개사 목록 확인
   - 공동중개사 추가/제거
   - 공동중개사 연락처 관리

3. **소유주 관리**:
   - 소유주 정보 확인
   - 소유주 변경 (필요한 경우)

### 4.3 모바일 대응

1. **반응형 레이아웃**:
   - 모바일에서는 사이드바가 햄버거 메뉴로 변환
   - 테이블 형태의 데이터는 카드 형태로 재구성
   - 폼 요소들은 수직으로 배치

2. **터치 최적화**:
   - 버튼 및 입력 요소 크기 확대
   - 스와이프 제스처 지원 (매물 사진 갤러리 등)

3. **오프라인 지원**:
   - 주요 데이터 로컬 캐싱
   - 오프라인 상태에서 기본 조회 기능 유지

## 5. 기술적 구현 세부사항

### 5.1 상태 관리

- **전역 상태**: React Context API를 활용
  - `AuthContext`: 인증 상태 및 사용자 정보
  - `PropertyContext`: 매물 목록 및 선택된 매물
  - `MatchingContext`: 매칭 관련 상태
  - `CodeContext`: 코드 테이블 데이터 (매물종류, 진행상태, 거래유형)

- **로컬 상태**: 컴포넌트 내 `useState` 활용
  - 폼 입력값
  - UI 상태 (모달 표시 여부, 탭 선택 등)

### 5.2 API 통신

- **Supabase 클라이언트** 활용
  - 데이터베이스 CRUD 작업
  - 인증 처리
  - 스토리지 관리

- **React Query** 활용
  - 데이터 캐싱
  - 자동 리프레시
  - 로딩/에러 상태 관리

### 5.3 성능 최적화

- **코드 스플리팅**: React.lazy 및 Suspense 활용
- **메모이제이션**: React.memo, useMemo, useCallback 활용
- **이미지 최적화**: 지연 로딩, 적절한 크기 조정
- **가상 스크롤**: 대량의 목록 데이터 처리 시

### 5.4 보안 강화

- **인증**: Google OAuth + Supabase Auth 조합
- **권한 제어**: 역할 기반 접근 제어 (RBAC)
- **데이터 보호**: Supabase RLS (Row Level Security) 활용
- **입력 검증**: 클라이언트 및 서버 측 유효성 검사

## 7. 핵심 컴포넌트 상세 명세

### 7.1 PropertyCard

- **목적**: 매물 목록에서 각 매물의 요약 정보 표시
- **속성**:
  - `property`: 매물 객체
  - `onClick`: 클릭 핸들러
- **기능**:
  - 매물 기본 정보 표시 (이름, 위치, 가격 등)
  - 매물 상태에 따른 뱃지 표시
  - 클릭 시 상세 페이지로 이동

### 7.2 PropertyFilter

- **목적**: 매물 검색 및 필터링 UI
- **속성**:
  - `onFilter`: 필터 적용 핸들러
  - `initialFilters`: 초기 필터 값
- **기능**:
  - 매물종류, 진행상태, 거래유형 선택
  - 가격대, 지역 필터링
  - 텍스트 검색
  - 필터 초기화 및 적용

### 7.3 ManagerAssignment

- **목적**: 매물의 담당자 배정 및 변경
- **속성**:
  - `propertyId`: 매물 ID
  - `currentManager`: 현재 담당자
  - `onAssign`: 담당자 배정 핸들러
- **기능**:
  - 현재 담당자 표시
  - 새 담당자 선택
  - 변경 사유 입력
  - 변경 이력 조회

### 7.4 CoBrokerManagement

- **목적**: 매물의 공동중개사 관리
- **속성**:
  - `propertyId`: 매물 ID
  - `coBrokers`: 현재 공동중개사 목록
  - `onUpdate`: 업데이트 핸들러
- **기능**:
  - 공동중개사 목록 표시
  - 공동중개사 추가/제거
  - 연락처 관리

### 7.5 GoogleLoginButton

- **목적**: Google 계정으로 로그인
- **속성**:
  - `onSuccess`: 로그인 성공 핸들러
  - `onError`: 로그인 실패 핸들러
- **기능**:
  - Google OAuth 인증 처리
  - 도메인 제한 (@the-realty.co.kr)
  - 로그인 상태 관리
